We are building an AI-powered crop recommendation and advisory platform designed specifically for marginal farmers in Assam and the Northeast, a region where access to agronomists is scarce, connectivity is unreliable, and one wrong cropping decision can devastate a family's income for an entire season. The system works across three layers — a Flutter mobile app for farmers and field workers that operates fully offline and syncs when connectivity returns, a Next.js web dashboard for field agents and district officers to manage soil data, publish hyperlocal pest alerts, and monitor crop health across their assigned regions, and a Python FastAPI backend that orchestrates a machine learning model trained on ICAR soil suitability data to generate field-specific, ranked crop recommendations based on real NPK and pH values logged for each individual plot. When a farmer selects a crop, the system immediately surfaces an economic viability score showing estimated input costs, expected yield, and projected profit per bigha calculated against live Assam mandi prices. A conversational AI advisor lets farmers ask questions in Assamese and receive personalized guidance grounded in their own soil profile, their village's active pest alerts, and the current season — making the system feel less like a database and more like a knowledgeable neighbour who knows their land.

tech stack :
Next.js with Zustand and TanStack Query is a solid combination — Zustand for lightweight global state like the logged-in user and role, TanStack Query for all server data fetching with caching and background refetch. Shadcn with Tailwind will let you build the dashboards fast without fighting a component library.
Flutter is the right call for the farmer app. Offline-first with Hive, good performance on mid-range Android, single codebase for Android and iOS.
FastAPI is well suited here — async by default, fast to write, and clean for exposing your ML model as an endpoint. Make sure your ML inference endpoint is separate from your CRUD endpoints in structure so you can swap model versions cleanly.
Supabase is doing a lot of good work for you — auth, Postgres, storage, and RLS all in one place. Lean on Row Level Security hard so you are not writing role checks in your FastAPI layer redundantly.
Celery with Upstash Redis is the right pattern for background tasks like weekly email reports, sync queue processing, and scheduled mandi price updates. Upstash is a good choice specifically because it is serverless Redis with a free tier that will survive the hackathon without configuration overhead.
